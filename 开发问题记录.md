### 3.1 配置类问题

#### [配置-001] pom.xml jjwt依赖版本冲突

**基本信息**
- 发现时间：2026-04-14
- 解决时间：2026-04-14
- 状态：✅ 已解决
- 责任人：AI辅助

**问题分类**
- 所属模块：后端框架
- 所属任务：后端认证模块
- 问题类型：配置问题

**问题描述**

错误信息：
```
Dependency 'jjwt-impl' in scope runtime does not match artifact dependencies
```

复现步骤：
1. 在pom.xml中添加jjwt依赖
2. 设置`<version>0.12.5</version>`
3. 为jjwt-impl和jjwt-jackson添加`<scope>runtime</scope>`
4. Maven刷新后显示版本冲突红线

**根本原因**
> jjwt-impl和jjwt-jackson设置为runtime scope后，与jjwt-api产生了版本解析冲突。JJWT 0.12.x系列要求这三个依赖保持相同的scope。

**解决方案**
> 1. 在properties中添加版本变量：`<jjwt.version>0.12.5</jjwt.version>`
> 2. 所有jjwt依赖使用`${jjwt.version}`引用版本
> 3. 移除runtime scope设置

```xml
<properties>
    <jjwt.version>0.12.5</jjwt.version>
</properties>

<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-api</artifactId>
    <version>${jjwt.version}</version>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-impl</artifactId>
    <version>${jjwt.version}</version>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-jackson</artifactId>
    <version>${jjwt.version}</version>
</dependency>
```

**预防策略**
> 1. 使用Maven版本变量统一管理依赖版本
> 2. 避免随意添加scope设置
> 3. 添加新依赖后先运行`mvn clean install`验证

**参考资料**
- [JJWT官方文档](https://github.com/jwtk/jjwt)
- [Maven依赖作用域](https://maven.apache.org/guides/introduction/introduction-to-dependency-mechanism.html#Dependency_Scope)

### 3.2 代码类问题

#### [代码-001] 前端request路径引用错误

**基本信息**
- 发现时间：2026-04-14
- 解决时间：2026-04-14
- 状态：✅ 已解决
- 责任人：AI辅助

**问题分类**
- 所属模块：前端框架
- 所属任务：认证API封装
- 问题类型：代码问题

**问题描述**

错误信息：
```
[plugin:vite:import-analysis] Failed to resolve import "./request" from "src/api/auth.js"
```

复现步骤：
1. 创建api/auth.js文件
2. 使用`import request from './request'`引用
3. 实际request.js在utils目录
4. Vite构建时报路径错误

**根本原因**
> 引用路径错误，request.js在utils目录，而非api目录

**解决方案**
> 使用绝对路径引用：
> ```javascript
> import request from '@/utils/request'
> ```

**预防策略**
> 1. 统一使用@/别名引用src目录
> 2. 创建文件时注意目录结构
> 3. 使用IDE的路径提示功能

**参考资料**
- [Vite路径别名配置](https://vitejs.dev/config/shared-options.html#resolve-aliases)

#### [代码-002] 前端静态资源引用缺失

**基本信息**
- 发现时间：2026-04-14
- 解决时间：2026-04-14
- 状态：✅ 已解决
- 责任人：AI辅助

**问题分类**
- 所属模块：前端框架
- 所属任务：主布局
- 问题类型：代码问题

**问题描述**

错误信息：
```
Failed to resolve import "@/assets/logo.png"
```

复现步骤：
1. 在MainLayout.vue中使用logo.png
2. 引用路径`@/assets/logo.png`
3. 但assets目录为空，无此文件
4. Vite构建时报资源缺失

**根本原因**
> Logo图片文件不存在，引用了未创建的静态资源

**解决方案**
> 使用Element Plus图标替代：
> ```vue
> <el-icon><Monitor /></el-icon>
> ```

**预防策略**
> 1. 使用系统图标代替图片资源
> 2. 创建引用资源前先确认文件存在
> 3. 使用占位符或默认图标

**参考资料**
- [Element Plus图标库](https://element-plus.org/zh-CN/component/icon.html)

#### [代码-003] 密码加密方式变更

**基本信息**
- 发现时间：2026-04-14
- 解决时间：2026-04-14
- 状态：✅ 已解决
- 责任人：AI辅助

**问题分类**
- 所属模块：后端认证
- 所属任务：用户登录
- 问题类型：代码问题

**问题描述**
- 原设计：使用BCryptPasswordEncoder加密密码
- 实际需求：使用明文验证

**根本原因**
> 用户要求简化处理，直接使用明文验证

**解决方案**
> 1. 移除BCryptPasswordEncoder Bean
> 2. 修改UserServiceImpl中的密码验证逻辑：
> ```java
> // 原来：passwordEncoder.matches(password, user.getPassword())
> // 改为：
> password.equals(user.getPassword())
> ```
> 3. 修改数据库初始化SQL，使用明文密码
> ```sql
> INSERT INTO sys_user VALUES ('admin', 'admin123', ...)
> ```

**预防策略**
> 1. 生产环境应使用加密存储
> 2. 明文存储需充分评估安全风险
> 3. 记录配置变更决策

**参考资料**
- [Spring Security密码编码](https://docs.spring.io/spring-security/reference/servlet/authentication/passwords/password-encoder.html)
